# 流水账 版本号规则

> 本文件供后续开发者接手时参考，描述项目的**两套版本号体系**及其更新规则。

---

## 一、开发迭代版本号（内部 / 显示给用户）

格式：`β.X.YYY` 或正式发布后的 `A.B.CCC`

### 当前阶段：γ（GitHub Release 阶段）

```
γ         ← 当前版本
```

**规则：**

| 位置 | 含义 | 何时递增 |
|------|------|---------|
| `γ` | 阶段标识 | GitHub 开源发布版本 |

**迭代序号自动递增规则（供 AI 接手时遵守）：**
- 每次修复 bug 或新增功能，末位序号 +1（如 `.006` → `.007`）
- 不需要人工确认
- 如果开发者明确说"这是一次大版本"，才更新中间轮次（`.0` → `.1`）并重置序号为 `.001`

### 正式发布后阶段

```
1.0.001   ← 正式版第 1 次小更新
1.0.015   ← 第 15 次小更新
1.1.001   ← 大版本更新后（开发者确认）
2.0.001   ← 重大重构（开发者确认）
```

## 二、打包版本号（pubspec.yaml / Android 构建号）

Flutter 的 `pubspec.yaml` 中的 `version` 字段格式为：

```
version: MAJOR.MINOR.PATCH+BUILD_NUMBER
```

**对应 APK 中的：**
- `versionName`（用户可见）= `MAJOR.MINOR.PATCH`，例如 `1.0.0`
- `versionCode`（内部递增整数）= `BUILD_NUMBER`，例如 `6`

### 当前值

```yaml
version: 1.0.0+6
```

即：用户在应用市场或 APK 信息中看到 `1.0.0`，内部构建号为 `6`。

### 更新规则

| 改动类型 | 怎么更新 | 示例 | 需人工确认？ |
|---------|---------|------|------------|
| Bug 修复 / 小功能 | `BUILD_NUMBER` +1，前三位不变 | `1.0.0+6` → `1.0.0+7` | ❌ 自动 |
| 功能批次积累（每 ~5 次构建） | `PATCH` +1，`BUILD_NUMBER` 继续递增 | `1.0.0+9` → `1.0.1+10` | ❌ 自动 |
| 较大功能组（UI 重设计、新模块） | `MINOR` +1，`PATCH` 归零 | `1.0.5+20` → `1.1.0+21` | ✅ **开发者确认** |
| 重大重构 / 架构变更 | `MAJOR` +1，后两位归零 | `1.1.3+35` → `2.0.0+36` | ✅ **开发者确认** |

> **注意：** `BUILD_NUMBER` 只增不减，即使 `PATCH`/`MINOR` 重置，`BUILD_NUMBER` 也要继续累加。

### 操作示例

```yaml
# 修复1个bug后（自动）
version: 1.0.0+6  →  1.0.0+7

# 再修复3个bug后（自动）
version: 1.0.0+7  →  1.0.0+8  →  1.0.0+9  →  1.0.0+10

# 积累了5次小改动，适时更新 PATCH（自动判断）
version: 1.0.0+10  →  1.0.1+11

# 开发者说"这次重做了设置界面，算大版本"（需确认）
version: 1.0.3+15  →  1.1.0+16
```

## 三、两套版本号对应关系

```
pubspec.yaml         显示给用户（About 页面）
─────────────        ──────────────────────
1.0.0+6      ←→     β.0.006
1.0.1+11     ←→     β.0.011
1.1.0+16     ←→     β.1.001   (开发者确认大版本)
```

`BUILD_NUMBER` 与迭代序号保持一致：`+6` 对应 `.006`。

---

## 四、AI 接手时的操作清单

每次 AI 处理一批修改后，应自动完成以下更新：

1. **`pubspec.yaml`** → `BUILD_NUMBER` +1（`PATCH` 按规则判断是否更新）
2. **`settings_screen.dart`** About 页面 → 更新显示版本字符串（如 `β.0.006` → `β.0.007`）
3. **`settings_screen.dart`** `_changelog` → 在顶部新增本次变更条目
4. **`VERSION.md`** → 如有规则变化才修改，否则不动

**不需要** 开发者手动告知版本号，AI 自行递增即可。  
**需要**开发者确认的只有：`MINOR` 或 `MAJOR` 的升级，以及 `β` 轮次的更新。

## 五、文件位置速查

| 文件 | 作用 |
|------|------|
| `pubspec.yaml` | Flutter 构建版本，改这里 APK 的 versionName / versionCode 才会变 |
| `lib/screens/settings_screen.dart` | About 页面显示的版本字符串 + `_changelog` 更新日志 |
| `VERSION.md` | 本文件，版本规则说明 |
